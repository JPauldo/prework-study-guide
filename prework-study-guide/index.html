HTML

  • <head>: contains information about the webpage
    • <title>: defines the title for a webpage
    • <style>: defines the CSS styles associated with a website
    • <base>: defines the base URL for a webpage
    • <meta>: elements that contain information about the page that is used by the browser
    • <link>: connects an external resource to the HTML document
      • rel: specifies the relationship between the current document and the linked document resource
      • href: points to the URL for the link
      • charset: sets character encoding
      • http-equiv: sets an HTTP header for content
      • viewport: sets the default view for a webpage
    • <script>: embeds client-side scripts in the HTML document
  • <body> element represents the visible content shown to the user
    • <h1-h6>: represents levels of prominent titles and subtitles of heading that are used to organize a page
    • <p>: represents a paragraph or block of text
    • <ul>: represents a list of items, where the order in which it is displayed matters
    • <ol>: represents a list of items, where the order fundamental does not matter when displayed
    • <li>: represents the item of a list
    • <img>: contains information about images that are displayed on the webpage
      • src: defines the location, or source, of the image file
      • alt: contains a text string that describes the appearance and functionality of the image
    • <br>: creates a line of empty space, or a line break, between two blocks of content
    • <a>[anchor]: creates links to the same webpage or other webpages
    • <div>: creates a division in the page
    • <span>: creates an inline division

CSS

  • Inline CSS: a style attribute with a CSS rule to style a particular page element only
  • Internal Style Sheet: a set of CSS rules defined for the webpage in the head section of an HTML file
  • External Style Sheet: a file that holds the set of CSS rules for an HTML file, linked in its head element
  • Selector: a specified element or attibutes that are modified by a declaration block
  • Declaration: a rule comprised of a CSS property and its assigned value
    • CSS Box Model: a model used to demonstrate the components of every HTML element
      • margin: a property that indicates how much space we want around the outside of an element; use mofidiers top, bottom, left and/or right for more specifics
      • border: a property that wraps the content and any padding
      • padding: a property that adds space around the content inside an element; use mofidiers top, bottom, left and/or right for more specifics
      • content: an HTML element
    • width: a property that controls the horizontal length of an element
    • height: a property that controls the vertical length of an element
    • background-color: a property that sets the background color of an element
    • color: a property that sets the color of an element's text
    • font-size: a property that sets the size of the font
    • text-align: a property that sets the horizontal alignment of the inline-level content inside a block element or table-cell box
    • display: a property that helps to determines the type of box that is generated for an element; use with float and position for best results
    • box-shadow: a property that adds shadow effects around an element's frame
  • Separation of concerns: a key design principle that refers to separating code into different files based on use
  • Do Not Repeat Yourself (DRY): a coding principal that encourages reuse/sharing to reduce written/maintained line count

Git

  • git status: checks what branch we are currently on
  • git add [file]: adds a file as it looks now to your next commit
    • -A: all files
  • git restore [file]: discards changes for a file in working directory
  • git checkout [branch-name]: switches to an existing branch
    • -b: creates a new branch and switches to it
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git pull origin [branch-name]: fetch and merge any commits from the tracking remote branch
  • git push origin [branch-name]: transmit local branch commits to the remote repository branch

JavaScript

  • var: a keyword used to declare a variable
    • variable: a named container that allows us to store data in our code
    • control flow: the order in which a computer executes code in a script
  • console.log: outputs a message to the web console based on an argument
    • console: a test environment that developers use to check out their code
    • method: a set of instructions that can be executed by a computer
    • log(): a method that outputs whatever we add inside the parentheses to the console
  • if: a statement that is executed if the conditional is met
  • else-if: a statement that is executed if all prior conditional(s) are not met
  • else: a statement that is executed if all prior conditional(s) are not met
    • truthy: non-falsy values, including but not limited to the Boolean value of true
    • falsy: 0, -0, On, "", null, undefined, NaN, or the Boolean value false
    • ===: an operator that returns a Boolean value based on if the two values are equal
  • array: a single variable that is used to hold a group of data; typically holds similar data
    • length: a sub function that retrieves the number of items in a given array
  • for: a loop that iterates based on the conditions stated
  • function: a set of instructions that instructs the computer on how to perform a certain task
  • Math: a built-in object that has properties and methods for mathematical constants and functions
    • floor(): returns the largest integer less than or equal to x
    • random(): returns a pseudo-random number between 0 and 1
